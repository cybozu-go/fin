# cksumvfs

Package that statically links the SQLite Checksum VFS shim.

## File Structure

| File | Description |
|---|---|
| `cksumvfs.c` | Official SQLite Checksum VFS implementation (obtained from upstream) |
| `cksumvfs.go` | cgo binding (statically linked with `-DSQLITE_CKSUMVFS_STATIC`). `Register()` calls `sqlite3_register_cksumvfs()` to activate the VFS shim. |
| `sqlite3.h` | SQLite amalgamation header (generated by the procedure below) |

## Updating sqlite3.h

`sqlite3.h` is generated from an amalgamation build of the SQLite source.
The version must match the SQLite version bundled with `go-sqlite3`.

### Steps

```bash
# 1. Get the SQLite version bundled in go-sqlite3
# If the module is already in the local cache (e.g. after running go build), use:
SQLITE_VERSION=$(sed -n 's/^#define SQLITE_VERSION *"\([^"]*\)".*/\1/p' \
  "$(go env GOMODCACHE)/$(go list -m -f '{{.Path}}@{{.Version}}' github.com/mattn/go-sqlite3)/sqlite3-binding.h")

# 2. Check out that version in the SQLite source repository
SQLITE_SRC=<path/to/sqlite>
git -C "$SQLITE_SRC" checkout "version-$SQLITE_VERSION"

# 3. Run configure to generate the Makefile
(cd "$SQLITE_SRC" && ./configure --quiet)

# 4. Generate sqlite3.h
make -C "$SQLITE_SRC" sqlite3.h

# 5. Copy it into this package
cp "$SQLITE_SRC/sqlite3.h" internal/cksumvfs/sqlite3.h
```

### Verifying the Version

```bash
grep "^#define SQLITE_VERSION " internal/cksumvfs/sqlite3.h
```

The version must match the SQLite version in `sqlite3-binding.h` of the `github.com/mattn/go-sqlite3` module pinned in `go.mod`.
